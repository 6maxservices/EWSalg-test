EWS V2.0 Documentation

Official technical documentation, engine logic, and verification reports.

1. Core Logic (5-Layer Engine)

Layer 1: Robust Baseline

Calculates "normal" state using Median and Dispersion (MAD/IQR) over a 12-week sliding window.

Layer 2: Point Anomaly

Uses EWMA to smooth data and calculates a Z-Score against the baseline.

Layer 3: Drift Detection

Uses CUSUM to detect slow, persistent shifts that a Z-score might miss.

Layer 4: Adaptive Thresholds

Uses Conformal Prediction to set limits based on historical residuals.

Layer 5: Health Score

Aggregates signals into a single 0-100% readiness metric.

Field Calculations

Field Formula Description
S(t) α * X_t + (1 - α) * S_{t-1} EWMA Smoothed Value.
Z-Score 0.6745 * (S - Med) / Disp Deviation from median.
Conf% TotalWeeks / 20 Confidence level (Capped at 100%).
Thresholds Percentile(Residuals, α) Adaptive limits.

2. Calibration Parameters

EWMA Alpha (α)

0.1 (Stable) to 0.7 (Sensitive). Controls how much weight the latest number carries. Lower = Smoother, Higher = Faster Reaction.

Window Size

Number of weeks used for the baseline. 12 weeks (3 months) is standard for weekly metrics.

CUSUM k & h

Detects slow trends. k (Slack) filters noise; h (Threshold) triggers the DRIFT alert when accumulated change overflows.

Conformal α

Statistical certainty. 0.10 targets 90% certainty for Warnings; 0.02 targets 98% for Critical alerts.
